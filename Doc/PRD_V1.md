# **产品需求文档 (PRD): 基于 Qwen2.5-Omni 的实时人机交互与情绪理解系统 (Mac Studio 优化版)**

## **1 文档概述**

### **1.1 项目背景**

本项目旨在利用 Qwen2.5-Omni-3B 的原生多模态端到端理解能力，在 **M3 Ultra Mac Studio** 上构建一套高性能机器人视觉与听觉系统。系统不仅要识别"看到了什么"，更要理解"感受到了什么"，通过向模型输入视频帧与音频并配合精心设计的提示词，实现对单人及多人场景下复杂情绪与社交氛围的实时判别。

[https://modelscope.cn/models/Qwen/Qwen2.5-Omni-3B](https://modelscope.cn/models/Qwen/Qwen2.5-Omni-3B)

### **1.2 核心目标**

- **多维情绪理解**：利用模型对面部表情、肢体语态、语音语调的端到端多模态理解能力，实现情绪识别。
- **群组与氛围感知**：支持多人场景下的社交关系识别及环境氛围（如：热烈、紧张、沉闷）的宏观理解。
- **实时交互反馈**：利用 M3 Ultra 统一内存实现端到端 <400ms 的推断延迟，确保机器人交互的自然感。

## **2 核心功能需求**

### **2.1 实时视频流采集与输入**

- **流式视觉输入**：利用 **AVFoundation** 获取 1080p/30fps 原始流。
- **帧采样与输入准备**：从连续视频流中按策略选帧（均匀/关键帧/自适应），控制送入模型的帧数以平衡精度与延迟。
- **多人场景 ROI 裁剪**：利用 macOS Vision 框架检测画面中的人物，裁出各人物区域分别送入模型进行个体分析。

### **2.2 实时音频流采集与时间窗口对齐**

- **音频采集**：以 16kHz 采样率实时采集音频流。
- **时间窗口缓冲**：将视频帧与音频按推理窗口（约 1.5 秒）组织，确保同一窗口内的视觉与听觉信号在时间上对齐后一并送入模型推理。

### **2.3 模型驱动的综合理解**

模型的理解能力通过提示词工程（Prompt Engineering）引导，而非手动特征提取：

- **单人模式**：通过 prompt 引导模型深度剖析个体的情绪一致性（如：嘴在笑但眼神充满疲惫），输出结构化的情绪判定结果。
- **多人模式**：通过 prompt 引导模型理解人与人之间的社交距离（Proxemics）和互动反馈，判定当前的群体氛围（如：家庭争吵、商业谈判或朋友聚会）。
- **状态追踪**：跨推理窗口累积情绪变化趋势，检测突变（如：从平静突然转为焦虑）。

## **3 技术架构设计 (Apple Silicon 优化)**

### **3.1 系统总体数据流**

```
采集层 → 输入准备层 → 提示词构建 + 模型推理 → 后处理层
  ↑                                              ↓
              pipeline (推理窗口调度与编排)
```

### **3.2 采集与输入准备层**

1. **视频采集**：通过 AVFoundation 获取 1080p/30fps 视频流。
2. **音频采集**：以 16kHz/单声道采集音频流。
3. **时间窗口缓冲**：将连续的视频帧和音频按 1.5s 窗口切片，供推理管道周期性取用。
4. **帧采样**：从每个窗口的 ~45 帧中选取 ~4 帧，平衡分析精度与推理延迟。
5. **ROI 裁剪（多人场景）**：利用 macOS Vision 框架检测人体区域，裁出各人物的画面子区域。

### **3.3 提示词工程层（系统核心智能）**

Qwen2.5-Omni-3B 是端到端多模态模型，面部表情识别、肢体动作理解、语音情绪分析、跨模态融合均由模型内部通过 Attention 机制自动完成。系统的核心智能在于 **如何向模型提问**：

1. **系统角色设定**：定义模型作为情绪分析专家的基础人设与行为约束。
2. **任务指令模板**：针对单人情绪分析、多人氛围判定等不同任务，设计差异化的分析指令。
3. **结构化输出约束**：通过 JSON Schema 约束模型输出格式，确保返回结果可被程序解析。

### **3.4 模型推理层：Qwen2.5-Omni-3B + transformers**

- **推理框架**：使用 transformers 库加载模型，通过 PyTorch MPS 后端在 Apple Silicon 上运行。
- **模型 Processor**：分辨率缩放、色彩空间转换、音频重采样等格式适配由 `Qwen2_5OmniProcessor` 内部完成，无需手动处理。
- **统一内存**：M3 Ultra 的 192GB 统一内存由硬件自动管理，CPU 与 GPU 共享地址空间，无需显式内存搬运。
- **推理输入**：conversation 格式的消息列表，包含 system prompt + 用户指令 + 视频帧 + 音频。
- **推理输出**：文本（JSON 格式的情绪/氛围分析结果）。

### **3.5 后处理层**

1. **响应解析**：将模型返回的 JSON 文本解析为结构化 Python 数据（dataclass）。
2. **状态追踪**：跨推理窗口累积情绪数据，检测趋势变化和突变事件。

## **4 业务场景 (Use Cases)**

| 场景           | 模型分析目标                                         | 机器人预期行为                       |
| ------------ | ---------------------------------------------- | ----------------------------- |
| **社交破冰机器人**  | 识别多名宾客的**面部表情**（尴尬/无聊）与**身体朝向**（是否背对人群）。       | 主动插入话题，或引导边缘位置的宾客参与互动。        |
| **高压环境心理干预** | 在审讯或咨询场景下，通过**微表情**与**语音震颤**识别用户是否处于极度焦虑或撒谎状态。 | 调整语速与光照，输出安抚性话语。              |
| **家庭情感中心**   | 识别家庭成员间的**社交氛围**（如：晚餐时的沉闷或喜悦），结合多人的**动作协调性**。  | 自动播放适配氛围的音乐，或在识别到争执迹象时进行温和提醒。 |
| **公共场所安全哨兵** | 识别特定人群的**异常奔跑**、**大幅度肢体冲撞**伴随**高分贝尖叫**。        | 判定为危险氛围，立即向管理中心发送带有情绪定性的告警。   |

## **5 非功能性需求**

- **延迟要求**：单人场景端到端 <400ms；多人场景（4-6人）<500ms。
- **鲁棒性**：在低照度或部分遮挡（如戴口罩）情况下，通过 prompt 引导模型加重对肢体动作和音频信号的关注。
- **隐私保护**：模型在 M3 Ultra 本地运行，所有推理在本地完成，不上传原始音视频数据，仅输出结构化的语义理解结果。
